## Key Features

### 1. Centralized Configuration

All environment variables are loaded through `lib/config.ts`:

```typescript lib/config.ts
export const AppConfig = {
  app: {
    name: "Discover Your Personality",
    description: "...",
  },
  toughTongue: {
    apiKey: process.env.TOUGH_TONGUE_API_KEY || "",
  },
  firebase: {
    apiKey: process.env.NEXT_PUBLIC_FIREBASE_API_KEY || "",
    // ... all Firebase config
  },
  admin: {
    token: process.env.ADMIN_TOKEN || "TTAI-STARTER-ADMIN-TOKEN",
    defaultToken: "TTAI-STARTER-ADMIN-TOKEN",
    isUsingDefaultToken: ...,
  },
};
```

### 2. Interactive MBTI Type Explorer

The landing page features:

- Cover image showcasing all 16 personality types
- Clickable type cards with character emojis
- Detailed modal dialogs with traits and strengths
- Beautiful UI with smooth animations

### 3. localStorage Data Persistence

The app stores data in browser localStorage:

- Personality test results with MBTI type
- All coaching session history
- Timestamps and session metadata

```typescript
import { STORAGE_KEYS } from "@/lib/constants";

// Save test result
localStorage.setItem(STORAGE_KEYS.PERSONALITY_TEST_RESULT, JSON.stringify(result));

// Get test result
const stored = localStorage.getItem(STORAGE_KEYS.PERSONALITY_TEST_RESULT);
const result = stored ? JSON.parse(stored) : null;
```

### 4. Admin Panel with Token Protection

Access the admin dashboard at `/admin` using your `ADMIN_TOKEN`:

**Features:**

- View app statistics (tests completed, sessions, storage usage)
- Export all data as JSON
- Clear individual data types or all data
- View configuration (scenario IDs)
- Security warnings for default token

**Default token**: `TTAI-STARTER-ADMIN-TOKEN` (⚠️ Change for production!)

### 5. Admin Security Banner

A yellow warning banner appears on all pages when using the default admin token, reminding you to set a custom `ADMIN_TOKEN` environment variable.

## Integration Examples

### Using Firebase Auth

```typescript Component Example
"use client";
import { useAuth } from "@/app/auth/AuthContext";

export default function MyComponent() {
  const { user, loading, signOut } = useAuth();

  if (loading) return <p>Loading...</p>;
  if (!user) return <p>Please sign in</p>;

  return (
    <div>
      <p>Welcome, {user.email}</p>
      <button onClick={signOut}>Sign Out</button>
    </div>
  );
}
```

### Listening to ToughTongue AI Events

```typescript Session Management
useEffect(() => {
  const handleMessage = async (event: MessageEvent) => {
    if (event.origin !== "https://app.toughtongueai.com") return;

    const { type, data } = event.data;

    if (type === "onStart") {
      console.log("Session started:", data.session_id);
    } else if (type === "onStop") {
      console.log("Session completed:", data.session_id);
      // Analyze the session
      await analyzeSession(data.session_id);
    }
  };

  window.addEventListener("message", handleMessage);
  return () => window.removeEventListener("message", handleMessage);
}, []);
```

### Calling API Routes

```typescript API Integration
// Analyze a session
const response = await fetch("/api/tough-tongue/sessions/analyze", {
  method: "POST",
  headers: { "Content-Type": "application/json" },

const analysis = await response.json();

// Get session details

```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Firebase: Error (auth/invalid-api-key)">
    **Solution:** - Double-check all Firebase config values in `.env.local` - Ensure you've enabled
    Email/Password and Google auth in Firebase Console - Restart dev server: `pnpm dev`
  </Accordion>

{" "}
<Accordion title="ToughTongue AI session not starting">
  **Possible causes:** - Invalid API key in `.env.local` - Scenario ID not configured in
  `lib/constants.ts` - Browser blocking microphone permission **Solution:** - Verify
  `TOUGH_TONGUE_API_KEY` in Developer Portal - Check scenario IDs in `lib/constants.ts` - Allow
  microphone access when prompted
</Accordion>

{" "}
<Accordion title="Admin panel won't accept token">
  **Solution:** - Check `ADMIN_TOKEN` in `.env.local` - Restart dev server after changing
  environment variables - Default token is: `TTAI-STARTER-ADMIN-TOKEN`
</Accordion>

{" "}
<Accordion title="Data not persisting">
  **Explanation:** Data is stored in browser localStorage (per-browser, per-domain) **Note:** -
  Clearing browser data will delete saved results - Data doesn't sync across browsers - For
  production, consider adding database sync
</Accordion>

  <Accordion title="Build errors">
    **Solution:** ```bash rm -rf .next node_modules pnpm-lock.yaml pnpm install pnpm build ```
  </Accordion>
</AccordionGroup>
