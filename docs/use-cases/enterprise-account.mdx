# Enterprise Account Guide

**Deploy voice AI training across your organization with branded workspaces, team management, and enterprise-grade access control.**

An Enterprise Account enables teams to use ToughTongue AI together within a shared organizational workspace. You get a branded subdomain, member management with different permission levels, usage quotas, organization-wide analytics, and fine-grained control over who can access which scenarios.

**Last updated:** December 30, 2025
</Note>

## Permission Model Overview

Our permission model works like **Google Drive**:

<CardGroup cols={3}>
  <Card title="Access Control" icon="lock">
    Only explicitly added users can access your subdomain. No implicit access.
  </Card>
  
  <Card title="Content Ownership" icon="user-shield">
    Scenario creators control their content. Admins can't access unless shared.
  </Card>
  
  <Card title="Embedding Exception" icon="code">
    Embedded scenarios (iframe/API) allow access without org membership.
  </Card>
</CardGroup>

This separation ensures administrative oversight doesn't compromise content privacy. **Just like Google Docs, a Workspace admin can't read your documents unless you share them.**

## Your Organization's Landing Page

### Branded Subdomain

Your organization gets its own subdomain: **`{your-slug}.app.toughtongueai.com`**

Every scenario, course, and session lives under this subdomain, creating a cohesive branded experience for your team.

### Customization

Customize through **Account Settings â†’ Organizations â†’ Edit (Pencil Icon)**:

<Tabs>
  <Tab title="Basic Information">
    - **Name:** Your organization's display name
    - **Brand URL (Slug):** Custom subdomain (lowercase, numbers, hyphens only)
    - **Description:** Markdown-supported description
    - **FAQ:** Markdown-supported frequently asked questions
    - **Public/Private Toggle:** For enterprise, keep this private
  </Tab>
  
  <Tab title="Brand Assets">
    - **Logo:** Displayed in header and landing page
    - **Cover Image:** Banner at the top of your landing page
    - **Social Links:** Icons linking to your resources
  </Tab>
  
  <Tab title="Content Selection">
    Choose which scenarios and courses to highlight. Members can always access your full library, but the landing page showcases what matters most.
  </Tab>
</Tabs>

### Access Control

Enterprise subdomain access is **segregated by design**. Only users you explicitly add can access your subdomain.

If someone tries to visit `yourcompany.app.toughtongueai.com` without being a member, they'll be denied access. Your scenarios, courses, and training data remain private to your organization.

## Member Management

### Understanding Roles

<CardGroup cols={3}>
  <Card title="Member (View)" icon="eye">
    **For:** Students, trainees, sales reps, employees
    
    **Can:**
    - Access shared resources
    - See personal quota and usage
    
    **Cannot:**
    - View org analytics
    - Manage members
    - Change org settings
  </Card>
  
  <Card title="Admin" icon="user-cog">
    **For:** L&D team, training managers, curriculum designers
    
    **Can:**
    - Manage members (add, remove, roles, quotas)
    - Access org-wide analytics
    - Configure landing page and branding
    
    **Example:** Instructional designer creating training scenarios for sales team
  </Card>
  
  <Card title="Owner" icon="crown">
    **For:** Organization creator
    
    **Can:** Same as Admin
    
    **Note:** Functionally identical to Admin
  </Card>
</CardGroup>

<Warning>
**Important:** Admins and Owners have organizational privileges but do **not** automatically have access to scenario content. Content access is always controlled by the scenario creator.
</Warning>

### Managing Quotas

Every member can be assigned a usage quota (minutes they're allowed to use). If you don't specify a quota when adding a member, they receive the default organizational quota.

Members can see their remaining quota on the billing page.

### Adding Members

<Tabs>
  <Tab title="Individual Add">
    Navigate to **Account Settings â†’ Members â†’ Add Member**
    
    Provide:
    - **Email** (required)
    - **Name** (optional)
    - **Role:** View, Read, or Edit
    - **Quota** (optional)
    
    If the email doesn't have a ToughTongue AI account, we'll create one automatically.
  </Tab>
  
  <Tab title="Bulk Upload (CSV)">
    Click "Bulk Upload" and upload a CSV:
    
    ```csv
    email,name,role,quota_minutes
    john@example.com,John Doe,view,25.0
    jane@example.com,Jane Smith,edit,100.0
    ```
    
    **Valid roles:** `view`, `read`, `edit`, `owner`
  </Tab>
</Tabs>

**Editing Members:** Update roles, adjust quotas, or remove members anytime via the edit icon in the members table.

## Scenario Access Control

Scenarios are the core content in ToughTongue AI. Enterprise gives you fine-grained control over who can access, edit, and interact with each scenario.

### Organization-Wide Permissions

When you create a scenario, you can set organization-wide permissions:

<CardGroup cols={3}>
  <Card title="Interact + View Own" icon="user">
    Practice and see only personal history.
    
    **Best for:** Training where privacy matters; each trainee sees only their own performance.
  </Card>
  
  <Card title="Interact + View All" icon="users">
    Practice and see everyone's sessions.
    
    **Best for:** Collaborative learning; classroom settings where reviewing peers adds value.
  </Card>
  
  <Card title="Interact + Edit + View All" icon="users-cog">
    Full collaborative access.
    
    **Best for:** Small teams building content together.
  </Card>
</CardGroup>

### Individual Access Grants

You can also choose **not** to grant organization-wide access. Then, use the **"Manage Access"** feature to add individual users by email and assign them a specific permission level (View, Read, or Edit).

<Note>
**Example Use Case:** You're running a leadership development program for senior managers. You don't want the general employee population to access these sensitive scenarios, so you create them as private and grant access only to program participants.
</Note>

If a scenario has org-wide "view" access but you grant a specific user "edit" access, that user can edit even though others can only view.

## Courses: Organizing Your Content

As your scenario library grows, organization becomes critical. **Courses let you group scenarios into structured learning paths.**

### What is a Course?

A curated list of scenarios, optionally combined with static content like videos or reading materials. Displayed on a dedicated "Course" page in your navigation.

### Why Use Courses?

<CardGroup cols={2}>

  </Card>

<Steps>

## API Integration

Enterprise resources are available via APIs.

### Personal Access Tokens (PAT)

Your API key for authenticating requests.

**Creating a PAT:**

<Steps>
  <Step title="Navigate to Developer Tools">
    Sidebar â†’ Developer Tools
  </Step>
  
  <Step title="Go to API Keys Tab">
    Find the API Keys section
  </Step>
  
  <Step title="Create New Token">
    Click "Create New Token"
    
    Name it descriptively (e.g., "LMS Integration" or "Analytics Script")
  </Step>
</Steps>

### The X-TT-ORG Header

Include your organization's ID to access org resources:

```bash
curl -X POST https://api.toughtongueai.com/api/public/test \
  -H "Authorization: Bearer YOUR_PAT_TOKEN" \
  -H "Content-Type: application/json" \
  -H "X-TT-ORG: YOUR_ORG_ID" \
  -d '{"input": "check"}'
```

Find your organization ID in Developer Tools.

<Warning>
**Without `X-TT-ORG` header:** Returns only your personal resources

**With `X-TT-ORG` header:** Returns organization resources (subject to your permissions)
</Warning>

### API Playground

Test endpoints without writing code in Developer Tools:

<Steps>
  <Step>Select your PAT token</Step>
  <Step>Choose an endpoint</Step>
  <Step>Fill in parameters</Step>
  <Step>Click "Send Request"</Step>

<Note>
The playground doesn't automatically include `X-TT-ORG`. Add it manually for org context.

### Scenario Access Tokens (SAT)

For embedding private scenarios in external applications (company intranet, customer portal).

<CardGroup cols={4}>
  <Card title="Lifespan" icon="clock">

  </Card>
  
  <Card title="Scope" icon="crosshairs">

  </Card>
  
  <Card title="Generation" icon="server">

  </Card>

<Warning>
**Important:** SATs cannot be used with `X-TT-ORG`. They're for single-scenario access only.

Contact us for guidance on programmatic access to private org scenarios.
</Warning>

### Available Endpoints

All endpoints support org context via `X-TT-ORG` and honor the permission system:

- **Scenarios:** List, get, create, update
- **Sessions:** List, get (with filters)
- **Balance:** Check wallet balance
- **Collections:** List, get courses
- **Purchases:** Create checkout sessions
